Philosophy

Andre Snyman's guiding principle

"We are not selling fear. We are selling safety. We are not a panic button. We are a community of people who care. Every word in this system — on WhatsApp, on the website, in every email — must reflect empowerment, confidence, and belonging."

The AI assistant is called AI Arnie — Andre's digital safety sidekick. Arnie is warm, calm, and direct. He never alarms. He never dramatises. He connects people to safety quietly and efficiently.

Webhook Rules

LOCKED — Do not change Ignore incoming requests ONLY when req.body.MessageStatus is present and truthy (these are Twilio delivery status callbacks). Never block on SmsStatus="received" — that is a genuine inbound member message.

Webhook URL

POST https://cyber-chaperone-r--ryfsny.replit.app/api/webhook/twilio

Set this in Twilio Console → Messaging → Sandbox Settings → "WHEN A MESSAGE COMES IN".

Message processing order

#CheckAction
0ICE contact detectionRoute to ICE reply handler (choices 1–4)
1Distress keywords (help, sos, emergency…)RED + auto-escalate to ICE
2Arrival keywords (arrived, i have arrived…)Complete trip
3Menu reset trigger (Hi, 0, Menu…)Show main menu
4Conversation state routingTRIP_FLOW → CLARIFICATION → CHECKIN → CYBER_CHAPERONE → MAIN_MENU
5START [from] to [dest] ETA [time]Create GREEN trip
6ETA drift ≥15 minCHECKIN flow; ≥45 min → auto-ICE escalation + AMBER
7Ambiguous movement with active tripAMBER + clarification menu
8Freeform trip-start / follow-upExisting parser

Public API Endpoints

These endpoints do not require operator login.

POST /api/webhook/twilio

Receives all inbound WhatsApp messages from Twilio. Set in Twilio sandbox settings.

POST /api/register

Registers a new member from the website form. Requires X-API-Key header matching REGISTER_API_KEY secret.

Body: { first_name, last_name, whatsapp_number, email, home_address, suburb, city, province, membership_tier, industry }

GET /api/healthz

Health check. Returns 200 if server is running.

POST /api/auth/login

Operator login. Body: { password }. Sets session cookie.

Authenticated API Endpoints

All require operator session (login first at dashboard).

GET /api/members?page=1&limit=50&search=&province=&city=

Paginated member list. Returns { data: [], pagination: { page, limit, total, pages } }.

GET /api/members/map

Lightweight GPS-only member list for radar map. Only returns members with coordinates.

GET /api/trips · POST /api/trips · PATCH /api/trips/:id

Trip CRUD. Status values: green | amber | red | completed | cancelled.

POST /api/broadcast

Send bulk WhatsApp message to a list of members. Max 500 per call. 80ms delay between sends.

Body: { message: string, recipients: [{ whatsappNumber, displayName }] }

GET /api/messages · GET /api/responders · GET /api/cases

Message inbox, responder network, and case management.

Broadcast / Bulk Send

The Radar page in the Situation Room is the operator tool for radius-based bulk messaging:

StepAction
1Open Radar page
2Toggle "Radius Mode" on
3Click any point on the map
4Adjust the radius slider (km)
5Panel shows all members inside the radius
6Filter by status (all / active / verified)
7Type your message and click Send
Limits: Max 500 recipients per broadcast. 80ms delay between each send to avoid Twilio rate limits. Currently requires WhatsApp Sandbox — members must have opted in.

How existing members were imported

How to import new members manually

-- Insert a single member (safe — ON CONFLICT ignores duplicates)
INSERT INTO members (first_name, last_name, display_name, whatsapp_number, member_status, source_batch)
VALUES ('John', 'Smith', 'John Smith', 'whatsapp:+27821234567', 'active', 'manual_import_2026')
ON CONFLICT (whatsapp_number) DO NOTHING;

Route enrichment (automatic, no API key needed)

When a trip is created, the system automatically: geocodes start + destination via Nominatim, calculates driving route via OSRM, stores the polyline, ETA, and checkpoint list on the trip record.

How to Rebuild from Scratch

This section exists so the system can survive Replit going offline, bankrupt, or unavailable. Any developer with this document and the GitHub repository can rebuild and redeploy the full system.

Step 1 — Clone the code

git clone https://github.com/[YOUR_GITHUB_ORG]/cyber-chaperone.git
cd cyber-chaperone
pnpm install

Step 2 — Set up a PostgreSQL database

Use any of: Neon (neon.tech — free tier), Supabase (supabase.com — free tier), Railway, or a local Postgres instance.

# Set the DATABASE_URL environment variable
export DATABASE_URL="postgresql://user:password@host:5432/dbname"

# Push the schema
pnpm --filter @workspace/db run push

Step 3 — Set all environment secrets

See the Environment Secrets section above. Set all 8 secrets as environment variables on your hosting platform.

Step 4 — Deploy

The system can run on any Node.js host: Railway, Render, Fly.io, DigitalOcean App Platform, or a plain VPS.

# Build the API server
pnpm --filter @workspace/api-server run build

# Start the server
PORT=8080 node artifacts/api-server/dist/index.mjs

# Build the frontend
pnpm --filter @workspace/situation-room run build
# Serve the dist/ folder as static files

Step 5 — Update Twilio webhook URL

In Twilio Console → Messaging → Sandbox (or your production WhatsApp sender) → update "WHEN A MESSAGE COMES IN" to your new server URL:

POST https://your-new-domain.com/api/webhook/twilio

Step 6 — Test

# Health check
curl https://your-new-domain.com/api/healthz

# Send "Hi" from Andre's WhatsApp to test the menu

GitHub Backup

The code must live on GitHub — independent of Replit. GitHub is free, permanent, and any developer in the world can access it.

How to set up (one-time — Andre does this)

  1. Go to github.com and create an account (or log in)
  2. Click New repository → name it cyber-chaperone → set to Private → Create
  3. Copy the repository URL (looks like https://github.com/yourname/cyber-chaperone.git)
  4. Tell your developer the repo URL — they can then clone it and have everything
  5. In Replit: open the Git panel (branch icon in sidebar) → add the GitHub URL as a remote → push

After setup — keeping it in sync

Replit automatically commits code changes. You or your developer can push to GitHub whenever you want a permanent backup:

git push github main
Never push secrets to GitHub. The .gitignore file already excludes .env files. Secrets stay in Replit's Secrets panel (or your hosting platform's env vars).
