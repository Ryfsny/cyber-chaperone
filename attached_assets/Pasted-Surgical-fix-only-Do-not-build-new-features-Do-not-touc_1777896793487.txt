ISSUE 1 — Trip-start parser is too fragile

Verified failure:
The live WhatsApp message:
"TEST LIVE — Leaving Fourways now heading to Rosebank Mall. ETA 14:40."

was incorrectly treated as an ETA follow-up.
It should have created a new trip.

Required fix:
The trip-start parser must detect a trip-start pattern anywhere inside the message body, not only at the beginning.

It must match messages like:
1. "Leaving Fourways now heading to Rosebank Mall. ETA 14:40."
2. "TEST LIVE — Leaving Fourways now heading to Rosebank Mall. ETA 14:40."
3. "Andre test: Leaving Fourways heading to Rosebank Mall ETA 14:40"
4. "Leaving from Fourways going to Rosebank Mall. ETA 14:40."

Minimum extraction:
- startLocation = Fourways
- destination = Rosebank Mall
- eta = 14:40

Trip-start detection must happen before ETA follow-up classification.

Expected action when trip-start is detected:
- close any previous active GREEN/AMBER/RED trip for that phone if applicable
- create a new active trip
- status = GREEN
- link the inbound message to the new trip
- save clean evidence note: "Trip-start message received from WhatsApp"
- send appropriate acknowledgement if outbound replies are already enabled
