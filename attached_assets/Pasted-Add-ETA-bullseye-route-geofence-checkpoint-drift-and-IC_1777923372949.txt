Core principle:
The member-provided ETA is the bullseye. The system must monitor variance from that ETA.

Trip route intelligence must store:
- originalMemberEta
- calculatedGoogleEta
- etaDriftMinutes
- routePolyline
- routeCorridorRadiusMeters
- checkpointList
- expectedCheckpointTimes
- lastKnownLocation
- lastKnownLocationTime
- lastMemberCheckInTime
- currentRouteConfidence: GREEN / AMBER / RED
- iceContactName
- iceContactPhone
- iceEscalationStatus

Route geofence:
Use the Google route polyline to create a route corridor.
When a member sends a location pin, compare the pin to the expected route corridor.
If near route and on time = GREEN.
If delayed or outside tolerance = AMBER.
If off-route + no reply or repeated missed check-ins = RED / human review.

Checkpoint logic:
Generate checkpoints based on route length and duration.
Each checkpoint must have:
- checkpoint name / area
- expected time
- tolerance window
- status: pending / confirmed / missed

ETA drift logic:
If ETA drift is minor, keep GREEN.
If ETA drift exceeds 15 minutes, send member check-in.
If ETA drift exceeds 30 minutes or repeats, mark AMBER.
If ETA drift + no reply, escalate to operator.
If distress words appear, mark RED immediately.

Member check-in message:
"Andre, Cyber Chaperone check-in.

Your ETA appears to have shifted.

According to our route calculation, you should be near:
[checkpoint/area]

Please reply:

1. I am okay
2. I am delayed
3. My ETA changed
4. I have stopped
5. I need help
6. I will send my location pin"

ICE contact rule:
Each member must have an In Case of Emergency contact.
The ICE contact must not receive normal GREEN trip updates.
The ICE contact is contacted only when escalation rules are met.

ICE escalation triggers:
- missed checkpoint + no member reply
- repeated ETA drift
- off-route concern + no reply
- RED distress event
- operator manually escalates

ICE message:
"Hi [ICE Name], this is Cyber Chaperone from eblockwatch.

[origin] → [destination].

We have not received the expected check-in.

Please try to contact [Member Name] and reply:

1. I reached them — they are okay
2. I reached them — they need help
3. I could not reach them
4. Please ask the Situation Room to call me"

Operator mirror must show:
- member name
- trip route
- original ETA
- calculated ETA
- ETA drift

- last known location

Privacy rule:
Do not send the full member conversation, dashboard notes, or private trip history to the ICE contact.
