Important scope correction before building:

Do not call projected trip icons "live position" unless the member has sent a current location pin or the Situation Room has confirmed live WhatsApp location.

Use these terms:

1. Actual last known position
- based on latest WhatsApp location pin
- solid marker

2. Projected route position
- estimated from departure time, route duration, ETA, and route polyline
- ghost marker / lighter marker
- label clearly as "projected"

Do not fake live tracking.

Build in this order:

PHASE 1 — Radar map only
- Full-screen Situation Room radar page
- Show active trips
- Show Green / Amber / Red / Blue status
- Show route line
- Show actual last known marker if available
- Show projected marker if route data exists
- Auto-refresh every 30 seconds
- Completed trips should be blue/faded
- No responder dispatch yet

PHASE 2 — Responder layer
- Add responder table
- Store trusted eblockwatch responder base/home location
- Show responders only on operator radar, not member-facing
- Allow filter by responder type
- Keep responder pins private and operator-only

PHASE 3 — Manual dispatch drafts
- On RED trip, show nearest responders
- Generate WhatsApp assist request drafts
- Do not auto-send
- Operator must confirm before any WhatsApp is sent
